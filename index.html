The Model

Practical rules

The Mindset

01

Never prompt directly in chat

Write in an editor. Read it twice. Dehumanize it -- you're configuring a system, not chatting with a colleague.

02

Scripting has evolved

Create skills, agents, and reusable context. If you're writing the same instructions twice, make it a skill.

03

Keep all context congruent

Don't tell the model how to be smart -- give it the right information. Your job is congruence, not instruction.

04

Think first, then prompt

Every wasted token is a wasted thought. Plan before you execute.
