# /metaphor

## Process
1. Identify the **abstract concept** to explain
2. Find a **concrete domain** the audience already understands
3. Map components 1:1 between domains
4. Verify the mapping doesn't break under scrutiny
5. Name the metaphor for easy reference

## Quality Checks
- [ ] Every key component has a mapping
- [ ] The metaphor is intuitive (no explanation needed for the concrete side)
- [ ] Edge cases don't contradict the mapping
- [ ] The metaphor is memorable (ideally visual or narrative)

## Example: Finding Memo

| AI Concept | Finding Nemo | Why it works |
|-----------|-------------|-------------|
| Agent/Model | Marlin | Intelligence that drives action |
| Subagent | Dory | Helps but limited memory |
| Human | Crush | Experienced, shows the way |
| Harness | EAC (current) | Infrastructure that enables the journey |
| Context | Ocean/Memory | The medium everything operates in |
| Goal | Finding Nemo | Clear objective driving all action |
