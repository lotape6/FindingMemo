# /plottwist

A plot twist reframes everything the audience thought they understood.

## Web Research Phase
Use WebSearch to find real-world twist material:
- **Recent breakthroughs**: Search for recent AI announcements or capability demonstrations that most of the audience won't know about yet. A real twist beats an invented one.
- **Counterintuitive data**: Search for recent studies or benchmarks with surprising results (e.g., "AI outperforms humans at X", "developers using AI write more bugs", etc.). Data-backed twists are harder to dismiss.
- **Industry shifts**: Search for recent pivots, acquisitions, or policy changes in the AI/dev tools space that reframe conventional wisdom.

## Rules
1. The twist must be **true** — not a trick, but a perspective shift
2. It should make the audience re-evaluate previous slides
3. Place it at ~40% of the deck (not the end — leave room to rebuild)
4. Signal it visually: change background, use a meme, pause

## Pattern
```
[Audience believes X]
  ↓
[Twist: X was actually Y all along]
  ↓
[Now re-read everything before with Y in mind]
```

## Example for this presentation
- Audience thinks: "This is a presentation about AI"
- Twist: "This presentation IS an AI agent execution. You're inside the context window right now."
- Rebuild: Now every slide is both content AND example
